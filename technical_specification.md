# Текстовое техническое задание и архитектурный план реализации библиотеки `react-native-ultra-bottom-sheet`

## 1. Введение и общие требования
Цель данного документа — спроектировать высокопроизводительную, расширяемую и легковесную библиотеку Bottom Sheet для React Native, лишенную рудиментарного JS-кода. Основной упор делается на перенос вычислительной логики, обработки жестов и геометрии на нативный уровень (**Kotlin + C++ для Android, Swift для iOS**) с использованием последних стабильных версий `react-native-gesture-handler` и `react-native-reanimated`.

### Ключевые архитектурные принципы:
* **Минимум JS-моста:** Исключение лагов из-за асинхронности прохода через JS Bridge. Все расчеты размеров, позиции и привязки к клавиатуре происходят синхронно в Native UI-потоке.
* **Синхронность с клавиатурой:** Плавное, безрывковое изменение высоты шторки в режиме `keyboard-avoid`.
* **Инверсия управления поиском:** Полная свобода кастомизации поля ввода без зашитой логики внутри ядра.
* **Иерархическая независимость Backdrop:** Шторка и подложка (Backdrop) располагаются как сиблинги (на одном уровне иерархии), что позволяет использовать нативные размытия (`BlurView`) и кастомные элементы.

## 2. Техническое задание (ТЗ) и функциональные требования

### 2.1. Механика геометрического позиционирования и Анимации
1.  **Тип анимации:** Допускается строго один тип анимации — появление/выдвижение снизу вверх (`translationY`). При закрытии шторка полностью уходит за нижнюю границу экрана.
2.  **Максимальное ограничение высоты (`maxHeight`):** Параметр, жестко лимитирующий предел раскрытия шторки. Хлястик (верхняя точка) фиксируется на этой высоте, и дальнейший свайп вверх блокируется или демпфируется (эффект резиновой ленты / rubber-banding).
3.  **Управление режимами расчёта высоты (Взаимоисключающий пропс):**
    * **Режим Dynamic Sizing (По умолчанию):** Шторка автоматически измеряет внутренний контент и подстраивает свою высоту под него. Если контент меньше максимально допустимого пространства экрана (с учетом Safe Area и клавиатуры), шторка принимает размер контента. Если больше — включается внутренний скролл.
    * **Режим Фиксированных Snap Points (`snapPoints`):** Массив чисел `[0.3, 0.5, 1]`, где значения представляют долю от доступной высоты экрана. При передаче этого пропса режим `Dynamic Sizing` автоматически отключается. `1` означает 100% доступной высоты с учетом верхних отступов и клавиатуры.
4.  **Кастомизация анимаций:** Возможность настройки параметров жесткости, затухания и скорости (`mass`, `damping`, `stiffness` или `duration`) для нативных анимаций открытия/закрытия.

### 2.2. Нативное взаимодействие с клавиатурой (`Keyboard-Avoid`)
1.  **Синхронизация кадра в кадр:** При появлении клавиатуры на Android и iOS изменение геометрии Bottom Sheet должно происходить строго одновременно с анимацией системной клавиатуры (использование `WindowInsetsAnimation` на Android и нативных обзерверов клавиатуры на iOS через Reanimated Keyboard API).
2.  **Фиксация верхней грани:** Верхняя часть открытого Bottom Sheet жестко закрепляется на своей позиции (или на уровне `maxHeight`), в то время как клавиатура «поджимает» нижнюю грань шторки вверх. Шторка не должна прыгать вверх целиком — `translateY` верхней кромки не меняется; уменьшается высота контейнера шторки и внутреннего content-wrapper.
3.  **Content-wrapper:** Весь прокручиваемый контент размещается в обёртке, высота которой анимированно пересчитывается по формуле `min(measuredContent, screen - topInset - keyboardHeight)`. Если бы клавиатура была прозрачной, позади неё был бы виден фон приложения, а нижняя грань шторки совпадала бы с верхней гранью клавиатуры.

### 2.3. Управление Жестами (`Gesture Handler`)
1.  **Отключение жестов:** Проп `enableHandlePanningGesture={false}` полностью отключает перетаскивание шторки за хлястик. Закрытие остаётся доступным через Backdrop/API.
2.  **Интеграция со скроллом:** Жест панорамирования шторки обрабатывается **только на хлястике** (handle). Область контента (`ScrollView` / `FlatList`) всегда отдаёт вертикальный свайп скроллу — закрыть шторку свайпом изнутри контента нельзя, даже если скролл находится в начале списка (`contentOffset.y === 0`).
3.  **Режим snap points:** Если шторка не раскрыта до максимальной точки, скролл контента заблокирован (`scrollEnabled={false}`), а изменение высоты выполняется перетаскиванием за хлястик. После достижения максимальной точки включается внутренний скролл; закрытие по-прежнему только через хлястик, backdrop или API.

### 2.4. Backdrop (Подложка)
1.  **Иерархия компонентов:**
    ```xml
    <View style={styles.container}>
       <BottomSheetBackdrop /> <!-- Слой 1 -->
       <BottomSheetModal />    <!-- Слой 2 (Сиблинг, не внутри бекдропа) -->
    </View>
    ```
2.  **Параметры по умолчанию:** Легкое нативное затемнение, реагирующее на прогресс открытия шторки (`opacity` завязана на `sharedValue` позиции шторки).
3.  **Кастомный Backdrop:** Возможность передать кастомный реакт-компонент в проп `backdropComponent` (например, для интеграции `@react-native-community/blur` для создания эффекта размытия).
4.  **Интерактивность:** Проп `onBackdropPress`. По умолчанию закрывает шторку. Должен предоставлять сигнатуру:
    ```typescript
    onBackdropPress={({ dismiss, ref }) => {
      // Пользовательская логика перед закрытием
      dismiss(); // Нативный вызов закрытия
    }}
    ```

### 2.5. Разделение бизнес-логики на 3 подвида
Для оптимизации производительности и чистоты кодовой базы, логика разделяется на три специализированных компонента, импортируемых отдельно.

#### Подвид 1: Классический Bottom Sheet (`BottomSheet`)
* Принимает любой пользовательский `children`.
* Самостоятельно вычисляет высоту и управляет нативным скроллом, если контент превышает лимиты экрана.

#### Подвид 2: Выбор элементов (`BottomSheetPicker` / `Selector`)
Специализированный компонент для списков, оптимизированный под работу с большими массивами данных через нативные структуры.
* **Режим Picker:** При клике на элемент (`onSelect`) шторка мгновенно нативно закрывается.
* **Режим Selector:** При клике элемент выбирается (меняет стейт), но шторка остается открытой.
* **Режим MultiSelector:** Позволяет выбрать несколько элементов (чекбоксы/теги), не закрывая шторку.
* *Оптимизация:* Использование концепта переиспользуемых ячеек на нативном уровне (аналог `FlashList`) во избежание лагов перерисовки JS.

#### Подвид 3: Компонент с интегрированным поиском (`BottomSheetSearchPicker`)
Расширение второго подвида с полноценным и изолированным управлением поисковой строкой.
* **Полное управление инпутом:** Компонент принимает кастомный TextInput через `renderTextInput`.
* **Управление через Ref:** Предоставление императивного интерфейса:
    ```typescript
    interface BottomSheetSearchRef {
      focus: () => void;
      blur: () => void;
      clear: () => void;
      setText: (text: string) => void;
    }
    ```
* **Автофокус (`autoFocus`):** Булев проп. Если `true`, фокус на инпут наводится в момент инициации анимации открытия шторки. На нативном уровне клавиатура начинает подниматься одновременно с выдвижением шторки, исключая промежуточные дёргания лэйаута.
* **Колбек для кастомных инпутов:** Если пользователь не передает кастомный инпут, библиотека предоставляет встроенный, возвращая ссылку на него через callback `onSearchRefReady={(ref) => ...}`.

## 3. Архитектура и технический стек

### Стек технологий:
* **C++ (JSI / React Native New Architecture):** Написание кастомных С++ стейт-глушителей для синхронизации потока жестов и инсетов клавиатуры (опционально, TurboModules).
* **Kotlin (Android):** Реализация кастомного `ViewGroup`, который слушает `WindowInsetsAnimation.Callback` и напрямую меняет размеры контейнера контента на уровне нативных LayoutParams, минуя стадию прохода Yoga Layout в JS.
* **Swift (iOS):** Использование `UIKeyboardWillShowNotification` / `UIKeyboardWillHideNotification` вместе с `UIViewPropertyAnimator` для обеспечения математической точности следования за кадром анимации операционной системы.
* **Reanimated v3 & Gesture Handler v2:** Драйверы жестов на UI-потоке.

### Схема взаимодействия компонентов (Layout Hierarchy):
```
[Screen Root View]
├── [Backdrop Контейнер (Абсолютное позиционирование)]
│     └── <DefaultBackdrop> ИЛИ <CustomBlurView> (Анимирует Opacity от 0 до 1 на основе SharedValue)
└── [Bottom Sheet Нативный Контейнер (Абсолютное позиционирование снизу)]
      ├── [Header / Handle View] (Обработчик жеста панорамирования, если включен)
      ├── [Search Input Slot] (Опциональное поле ввода с автофокусом)
      └── [Content Wrapper] (Анимированная высота: уменьшается при keyboard-avoid)
            └── [ScrollView / FlatList / children]
```

## 4. План реализации (Шаг за шагом)

### Этап 1: Проектирование нативного ядра (Core Core)
1.  **Разработка структуры Монорепозитория:** Разделение на `@ultra-sheet/core`, `@ultra-sheet/picker`, `@ultra-sheet/search`.
2.  **Нативные обертки (Kotlin/Swift):**
    * Создание базовых нативных вьюх, способных принимать размеры измеряемого контента напрямую из нативного дерева UI.
    * Реализация перехватчиков инсетов клавиатуры. На Android: имплементация `setWindowInsetsAnimationCallback`. На iOS: интеграция с безопасной зоной через `safeAreaInsets`.

### Этап 2: Интеграция Жестов и Логики Размеров (UI-поток Reanimated)
1.  **Реализация Dynamic Sizing:** Написание Worklet-функции в Reanimated, выполняющей нативный `measure` контейнера контента сразу после его рендеринга и присваивающей значение в `sharedValue` высоты.
2.  **Реализация логики `snapPoints`:** Написание математического распределения (маппинга) точек остановки жеста. Интеграция условий отключения динамического размера при наличии массива кастомных точек.
3.  **Обработка `keyboard-avoid`:** Написание формулы расчета высоты в UI-потоке:
    $$	ext{ActualContentHeight} = \min(	ext{MeasuredHeight}, 	ext{ScreenHeight} - 	ext{TopInsets} - 	ext{KeyboardHeight})$$
    Привязка этой формулы к изменению высоты нижней нативной рамки.

### Этап 3: Создание высокоуровневых модулей (Picker & Search)
1.  **Создание `BottomSheetPicker`:** Интеграция легковесного списочного движка. Написание оптимизированных колбеков закрытия для обычного Пикера и сохранения состояния для Мультиселектора.
2.  **Создание Поискового модуля:** Разработка Imperative API (Refs). Написание логики синхронного триггера `focus()` инпута одновременно с запуском анимации `translationY` шторки вверх при `autoFocus={true}`.

### Этап 4: Разработка Backdrop и Кастомизации
1.  **Связывание Backdrop с позицией шторки:** Экспорт `animatedPosition` из ядра шторки в качестве реактивного инпута для Backdrop.
2.  **Имплементация `onBackdropPress` API:** Передача нативных методов деструктуризации (`dismiss`) внутрь JS-колбека для предоставления контроля разработчику.

### Этап 5: Тестирование, Оптимизация производительности и Профилирование
1.  **Проверка FPS на Android (низкобюджетные устройства):** Профилирование через Android Studio Profiler (Systrace). Цель: 60 FPS при одновременном открытии клавиатуры и анимации шторки.
2.  **Проверка утечек памяти:** Тестирование пересоздания инпутов и рефов при многократном открытии/закрытии.
3.  **Выпуск стабильной версии:** Документирование API, подготовка примеров (Expo и CLI совместимых).
